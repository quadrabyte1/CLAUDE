\

\

When a developer invents a new code she will be expected to supply (this list subject to consensus):

- ☐

  A very specific symbolic name, following some naming convention(s) we will agree upon. For example: OCCURRENCE_POWER_SUPPLY_X_VOLTAGE_EXCEEDS_LIMIT

- ☐

  A description of what the new code means, in English initially. I would not expect that *these* codes will ever trigger a user-facing message directly. If the user sees a message as an indirect result of such an announcement, there will always be some kind of UI-related processing where the language issue can be handled.
